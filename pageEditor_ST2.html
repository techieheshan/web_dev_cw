Splash Screen

HTML Structure

The Splash Screen does not use the shared site template (the spec allows this, since it's meant to look different from the rest of the site). It has its own <main class="splash"> containing an overlay div, a content block with the title, mission statement, team list, a countdown area, and a Skip Intro link. A <meta http-equiv="refresh" content="4; url=home.html"> tag in the <head> handles the automatic redirect, as required by the spec.

CSS Techniques

The background photo, dark gradient overlay, and centred content use position: relative/ absolute layering. The loading indicator is a CSS-only spinning ring (border-top-color + a @keyframes rotation), not a GIF, per the spec's explicit requirement.

JavaScript Logic

A setInterval counts down from 4 and updates the on-page number every second. The Skip Intro button removes the pending <meta http-equiv="refresh"> element from the DOM (cancelling the automatic redirect) before its own link takes the user to the Home Page immediately.

SDG Action Impact Simulator

HTML Structure

Six <article class="action-card"> elements, each carrying a data-points attribute holding its impact score, plus an image, a title, a description, and a visible impact-points badge. A separate .impact-summary panel shows the running total, impact level, and feedback message.

CSS Techniques

Page-specific styling is embedded in a <style> block, as the spec requires for this page. A .selected class (toggled by JavaScript, not CSS) changes a card's border colour and adds a "Selected" label via ::after. A fixed-position .ais-bg div sits behind everything and swaps its background image via .level-medium/ .level-high classes.

JavaScript Logic

Clicking or pressing Enter/Space on a card toggles a selected class and recalculates the total from every selected card's data-points. The total is compared against two thresholds to pick one of three impact levels, each with its own message and background class — so the background image change is driven entirely by JavaScript, not a CSS-only trick, per the spec.

Content Page — "Bridging the Digital Divide"

HTML Structure

One <h1> and four <h2> sections, each with a unique id, linked from a simple top navigation list via anchor links.

CSS Techniques

Uses the shared css/content.css file (also used by Students 3 and 4's content pages) rather than page-specific embedded CSS, since this page's layout is a straightforward single column and didn't need bespoke styling. The "Go to Top" button is plain HTML/CSS, no JavaScript.

Design Rationale

The topic connects to the Home Page's second focus-area card. It focuses specifically on device and connectivity access, distinct from Student 1's content page on the exam-to-employment skills gap.

Challenges & Lessons Learned

Placeholder — rewrite in your own words

Draft points based on the actual build, to be rewritten as a personal account:

  • Getting the automatic meta-refresh redirect to work alongside a JavaScript countdown took some care — the two had to be kept in sync without one interfering with the other.
  • Deciding on fair impact-point values for each action card, and picking thresholds for Low/ Medium/High that felt meaningful rather than arbitrary.
  • Making the background-image change feel smooth rather than jarring when the impact level changes.

Accessibility Considerations

  • Alt text on every image across both pages, describing what's shown.
  • Keyboard access on the AIS action cards via tabindex="0" and an Enter/Space key handler identical in effect to a click.
  • Live region (aria-live="polite") on the Splash Screen's countdown so screen readers announce the countdown changing.
  • No motion-only cues — the AIS impact level is always shown as text, not just implied by the background image changing.
