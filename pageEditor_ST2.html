Splash Screen

Technical Description

The Splash Screen does not use the shared site template, since it's meant to look different from the rest of the site, though it still loads the same Poppins/Inter font pairing (reference 3) for visual continuity. It has its own <main class="splash"> containing an overlay div, a content block with the title, mission statement, team list, a countdown area, and a Skip Intro link. A <meta http-equiv="refresh" content="4; url=home.html"> tag handles the automatic redirect. The background photo (reference 1), dark gradient overlay, and centred content use position: relative/absolute layering. The loading indicator is a CSS-only spinning ring (border-top-color plus a @keyframes rotation), not a GIF. A setInterval (reference 4) counts down from 4 and updates the on-page number every second; the Skip Intro button removes the pending meta refresh element from the DOM before its own link takes the user to the Home Page immediately.

Accessibility

The countdown region has role="status" and aria-live="polite", so a screen reader announces each second change rather than staying silent while the page auto-redirects. The Skip Intro control is a real <a> link, reachable and activatable by keyboard alone.

Action Impact Simulator

Technical Description

Six <article class="action-card"> elements, each carrying a data-points attribute holding its impact score, plus an image (reference 2), a title, a description, and a visible impact-points badge. Page-specific styling is embedded in a <style> block. A .selected class, toggled by JavaScript rather than CSS, changes a card's border colour. A fixed-position .ais-bg div sits behind everything and swaps its background image via .level-medium/.level-high classes (both reference 2; the default low-level background is reference 1). Clicking or pressing Enter/Space on a card toggles a selected class and recalculates the total from every selected card's data-points; the total is compared against two thresholds to pick one of three impact levels, so the background image change is driven entirely by JavaScript, not a CSS-only trick.

Accessibility

Each action card has tabindex="0" and an Enter/Space key handler with the same effect as a click. Rather than adding role="button" to an <article> that contains a heading, the selected state is communicated with a dynamically updated aria-label instead. The impact level is always shown as visible text, not just implied by the background image. All six card images have descriptive alt text.

Content Page: Bridging the Digital Divide

Technical Description

One <h1> hero, a pill-style jump-nav, and six <section> elements with unique ids: four alternating image/text "story blocks" (images credited in reference 2), a centred statement-quote, an action-tile grid, and a closing outcome section. Uses page-specific embedded <style> CSS rather than the shared css/content.css file used by Students 3 and 4, since this page needed its own layout: alternating "zigzag" story blocks with large numbered labels behind each image, instead of a sticky sidebar.

Accessibility

The jump-nav links are ordinary anchor links rather than JavaScript click handlers, so they work with the keyboard and with JavaScript disabled. Each of the six sections has scroll-margin-top so jumping to it doesn't hide the heading behind the sticky site header. All four article images have descriptive alt text.

Challenges and Lessons Learned

Getting the automatic meta-refresh redirect to work alongside a JavaScript countdown took some care, since the two had to be kept in sync without one interfering with the other — the Skip Intro button had to actively remove the pending meta refresh rather than just navigate away, otherwise a slow click could still get overridden by the redirect firing underneath it. Deciding on fair impact-point values for each action card, and picking thresholds for Low/Medium/High that felt meaningful rather than arbitrary, took a few passes of testing different combinations of selected cards to see where the boundaries actually felt right. Making the background-image change feel smooth rather than jarring when the impact level changes was mostly a matter of adding a CSS transition on the background rather than letting it snap instantly.

Accessibility Considerations

Across the Splash Screen, Action Impact Simulator and Content page, the main accessibility focus was making sure JavaScript-driven interactions still work without a mouse and are still announced correctly. The Splash Screen's countdown region has role="status" and aria-live="polite", so a screen reader announces each second change as it happens rather than staying silent while the page auto-redirects, and the Skip Intro control is a real <a> link rather than a div with a click handler, so it's reachable and activatable by keyboard alone. On the Action Impact Simulator, each action card has tabindex="0" and an Enter/Space key handler with the same effect as a click; rather than adding role="button" to an <article> that already contains its own heading (which would create a misleading ARIA role), the selected state is communicated with a dynamically updated aria-label instead, and the impact level is always shown as visible text alongside the background-image change, not communicated by colour alone. On the Content page, the jump-nav uses ordinary anchor links rather than JavaScript click handlers, so it keeps working with the keyboard and even with JavaScript disabled. All images across all three pages carry descriptive alt text. Together these choices mean the interactive features that make these pages distinctive don't come at the cost of excluding keyboard or screen-reader users.
