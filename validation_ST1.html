Validation Page — Student 1

Home Page validation report

Checked with the official W3C Nu Html Checker (validator.w3.org/nu/) against the page's live, publicly deployed URL, brightpath-one.vercel.app/home.html. Result: Pass — 0 errors, 0 warnings. The first check returned a real error: the three mission-card headings were marked up as <h3>, but appeared before any <h2> on the page, so the outline jumped straight from <h1> to <h3>. The fix was to add a proper <h2> ("Why we're doing this") introducing the cards section, which resolved the error without changing how the page looks.

Gallery Page validation report

Checked with the official W3C Nu Html Checker against brightpath-one.vercel.app/gallery.html. Result: Pass — 0 errors, 1 warning (Empty heading on <h3 id="modalTitle"></h3>, visible in the screenshot below). This warning is expected and acceptable: this heading is the extended-view modal's title, which starts empty in the HTML source and is filled in by JavaScript (modalTitle.textContent = ...) only once a user clicks a thumbnail. The page also initially failed for a real reason: the hidden modal's <img> placeholder had src="", which HTML doesn't allow. The fix was a tiny inline transparent placeholder image (data:image/gif;base64,...) that satisfies the validator and is invisible until JavaScript replaces it with the real photo on click.

Content Page validation report

Checked with the official W3C Nu Html Checker against brightpath-one.vercel.app/content_ST1.html. Result: Pass — 0 errors, 0 warnings. The lesson that carried across the Home and Gallery fixes: the validator doesn't just check for typos, it checks that the heading structure actually forms a logical outline, and that "hidden until JavaScript fills it in" elements still have to be valid HTML in their empty state, not just their final state.
